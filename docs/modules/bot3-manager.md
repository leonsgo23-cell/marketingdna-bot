# bot3.js — карта функций (менеджерский бот)

**Размер**: ~1053 строк  
**Роль**: интерфейс менеджера (Александр) для проверки и одобрения контента  
**Ключевое правило**: клиент НИКОГДА не получает контент без одобрения здесь

## Ключевые функции

| Функция | Строка | Что делает |
|---------|--------|-----------|
| `getSession(id)` | 24 | Загрузка сессии менеджера |
| `saveSession3(id, data)` | 31 | Сохранение сессии менеджера |
| `requireAuth(fn)` | 51 | Middleware: только авторизованный менеджер (BOT3_ACCESS_CODE) |
| `getSections(packageKey)` | 247 | Список разделов для проверки по типу пакета: photos/carousels/stories/covers + highlights+videos (Профи/Стандарт) |
| `showSection(ctx, sess)` | 253 | Показать раздел контента менеджеру |
| `showImageSection(ctx, sess, section)` | 264 | Показать визуальный раздел (фото/карусели) |
| `showVideos(ctx, sess)` | 295 | Показать видео на проверку |
| `getSectionUrls(data, section)` | 338 | Получить URL медиафайлов раздела |
| `showFinalApproval(ctx, sess)` | 457 | Финальное одобрение перед отправкой клиенту |

## Кнопки менеджера

| Callback | Что делает |
|---------|-----------|
| `send_free_{chatId}` | Одобрить бесплатный пакет → создаёт `.free_approved.trigger` |
| Одобрение разделов | Проверка по разделам → финальное `approved.trigger` → `deliverVisualPackage()` |

| `va_edit_{chatId}` | ✏️ Исправить сценарии — сбрасывает ВСЕ другие session-состояния, ставит `awaitingVideoScriptEdit`, просит фидбек → Sonnet переписывает → новый превью с кнопками |
| `et_hook_{i}_{chatId}` | ✏️ Изменить хук — спрашивает только хук, остальные поля сохраняются |

| `et_cta_{i}_{chatId}` | ✏️ Изменить CTA — спрашивает только CTA |

| `rscene_{videoIndex}_{chatId}` | 🔄 Переснять сцену — показывает список сцен, просит фидбек → AI выбирает сцены → Veo3 регенерирует |
| `regen_lib_{videoIndex}_{chatId}` | 🆕 Сгенерировать новое — вызывает `/force_generate_video`, запускает Veo3 минуя библиотеку |

## Лого клиента

| Команда | Что делает |
|---------|-----------|
| `/set_logo {chatId} [позиция]` | Сохраняет лого. Позиции: `br` (право-низ, по умолчанию), `bl`, `tr`, `tl`. После команды — прикрепить PNG/JPG |
| `/remove_logo {chatId}` | Удаляет лого клиента |
| `/add_reference {chatId}` | Принять видео-пример от менеджера. После команды — отправить видео (mp4/mov, до 90 сек, до 50 МБ). Паттерн применяется в Block7 |

Применяется автоматически ко всем изображениям и видео при показе менеджеру и доставке клиенту.

## Тестовые команды

```bash
/test_quality 71950950 v      # Тест качества: 1 штука каждого типа (quality.marker → maxPerSection:1)
/run_visual 71950950          # Запустить ПОЛНЫЙ визуал (все карусели/фото/сторис/обложки + 1 видео)
/run_visual 71950950 nv       # Полный визуал, без видео
/run_visual 71950950 nv qt    # Тест: 1 карусель + 1 фото + 1 сторис + 1 обложка, без видео
/debug_snapshot 71950950      # Диагностика done_snapshot: поля, промпты, qualityTest флаг, первые строки
/save_library 71950950        # Сохранить фото/видео клиента в общую библиотеку
/reset_client 71950950        # Полный сброс (удаляет visual.json, сохраняет done_snapshot)
/test_paid 71950950 v         # Тест Профи без Stripe (нужен BOT3_ACCESS_CODE)
/learning_stats               # Статистика самообучения: сколько правок накоплено, уроки
```

### Флаги /run_visual

| Флаг | Значение |
|------|---------|
| _(нет)_ | Полный пакет + 1 видео |
| `nv` | Полный пакет, без видео |
| `qt` | 1 штука каждого типа (quality test, maxPerSection=1) |
| `nv qt` | 1 штука каждого, без видео — **для тестов** |

**Важно:** `nv` и `qt` — независимые флаги, читаются в любом порядке. qualityTest из done_snapshot больше НЕ переносится автоматически — только явный `qt` флаг.

## Текстовый пакет через Bot3 (13.06.2026)

**Флоу одобрения текста:**
1. Визуал одобрен в Bot3 → `deliverVisualPackage` вызывает `deliverClientPackage`
2. `deliverClientPackage` строит HTML-страницу и **отправляет в Bot3** с кнопкой "📤 Отправить текстовый пакет клиенту"
3. Менеджер нажимает кнопку → ссылка уходит клиенту в Bot2
4. URL хранится в `{chatId}.text_url_wave{N}.json` до нажатия кнопки

**Кнопка:** `send_text_{chatId}_{waveNum}` — читает сохранённый URL, отправляет в Bot2, удаляет файл.

## /resend_scripts — повторная отправка сценариев с кнопками (24.06.2026)

```
/resend_scripts {chatId}
```

- Читает сценарии из `{chatId}.video_scripts_pending.json` (или `done_snapshot.json` как fallback)
- Повторно показывает превью с кнопками [✅ Запустить] [✏️ Исправить]
- Использовать когда: кнопки исчезли (нажал "Исправить", бот перезагрузился), или нужно одобрить после `/run_visual`

**Полный флоу видео (Стандарт/Профи):**
1. После картинок приходит превью сценариев с кнопками ✅ / ✏️
2. Если нужно исправить — пишешь фидбек → Sonnet переписывает → новый превью
3. Нажимаешь ✅ → Veo3 генерирует каждое видео (~7-10 мин)
4. Каждое видео приходит с кнопками (3 ряда):
   ```
   [✏️ Изменить хук]  [✏️ Изменить тему]  [✏️ Изменить CTA]
   [✏️ Изменить всё]  [🔄 Переснять сцену]
   [🎬 Версия без текста]
   ```
5. Каждая кнопка спрашивает только своё поле. Остальные поля сохраняются из предыдущего видео.
6. "Версия без текста" — присылает чистое видео (для ручного наложения текста в Instagram)

**Лимиты текста:**
- Хук: до 35 символов (1 строка, первые 4 сек)
- Тема: до 35 символов (1 строка, середина видео)
- CTA: до 66 символов (3 строки × 22 символа, до конца видео)

## /view_library — просмотр подходящих видео из библиотеки (25.06.2026)

```
/view_library {chatId}
```

- Вызывает `POST /library_matches` в visual.js
- visual.js читает videoScripts из visual.json (или done_snapshot), для каждого сценария извлекает теги (Claude Haiku), ищет совпадения (≥2 тегов) по всей библиотеке
- Для каждого сценария: показывает до 3 подходящих видео с тегами и количеством совпадений
- Если совпадений нет — показывает теги сценариев (чтобы понять почему нет матчей)
- Работает с любым `chatId` у которого есть videoScripts (visual.json или done_snapshot)
- Занимает 10-20 сек (AI-вызов на каждый сценарий)

**Когда использовать:** перед запуском генерации — посмотреть есть ли подходящие видео, не тратить Veo3. Или после нескольких клиентов — убедиться что библиотека накапливается и матчится.

**Автоматический режим:** при `/run_visual` система сама делает тот же поиск. Если матч ≥2 тегов — использует библиотеку (не запускает Veo3). Менеджер видит "📚 Из библиотеки" + кнопку "🆕 Сгенерировать новое".

## /resend_video — повторная отправка видео платного пакета (22.06.2026)

```
/resend_video {chatId} {index}
```

- Индекс: 0 = первое видео, 1 = второе (по умолчанию 0)
- Читает `localPath` из `{chatId}.results.json → videoData[index]`
- Повторно вызывает `notifyBot3SingleVideo` с уже готовым файлом
- Если `sendVideo` не прошёл (файл > 50MB) → автоматически пробует `sendDocument`

**Когда использовать:** если написало "Видео N готово" но сам файл не пришёл.

## /retry_free_slots — ретрай пропущенных слайдов карусели (21.06.2026)

Команда для ручного повторного запуска пропущенных слайдов карусели бесплатного пакета.

```
/retry_free_slots {chatId}
```

- Читает промпты из `visual_results/{chatId}.free_prompts.json`
- Определяет какие слайды (`carousel_N`) отсутствуют в `free_visuals.json`
- Сбрасывает флаг `carousel_notified` → после успеха карусель придёт заново
- Отправляет в Kie.ai только пропущенные слайды, ждёт результата
- Уведомляет менеджера о результате (успех / ещё пропущены)

**Когда использовать:** когда авто-ретрай (срабатывает автоматически после генерации) тоже не помог.

## /fix_html — пересобрать HTML из готовых файлов (15.06.2026)

```
/fix_html {chatId}
```

Пересобирает HTML-страницу клиента из уже готовых изображений без повторной генерации. Читает `free_visuals.json` и `free_photo.json`, вставляет пути в HTML.

## /reset_client — что удаляет / что сохраняет (15.06.2026)

**Удаляет:** results.json + все медиафайлы, triggers (кроме done_snapshot), pending, visual.json, visual_queue

**Сохраняет:**
- `{chatId}.done_snapshot.json` — скрипты (используется `/run_visual` и `/retry_paid`)
- `content_history/{chatId}.json` — история тем (защита от повторного контента)
- `photo_library/` и `video_library/` — общая библиотека

⚠️ visual.json теперь УДАЛЯЕТСЯ (раньше сохранялся). Это защита от просачивания старых промптов другого бизнеса. `/run_visual` пересоздаёт его из done_snapshot.

## Bot4 — финальная проверка перед отправкой клиенту (15.06.2026)

Отдельный бот (`bot4.js`) для финальной проверки перед доставкой клиенту.

**Флоу:**
1. Bot3: менеджер проверяет все разделы → нажимает "📤 Отправить клиенту"
2. Если `TELEGRAM_BOT4_TOKEN` задан → система пишет `{chatId}.bot4_review.trigger`
3. Bot4 подхватывает trigger → присылает менеджеру весь пакет: HTML-ссылка + все визуалы + кнопка
4. Менеджер нажимает **"📤 Отправить [Имя]"** → HTML обновляется финальными изображениями → клиент получает пакет

**Переменные Railway для Bot4:**
- `TELEGRAM_BOT4_TOKEN` — токен Bot4 (создать через @BotFather)
- `BOT4_MANAGER_CHAT_ID` — chatId менеджера (необязательно, Bot4 запомнит при /start)

**Активация:** написать `/start` в Bot4 — бот запомнит chatId менеджера.

Если `TELEGRAM_BOT4_TOKEN` не задан — "Отправить клиенту" в Bot3 доставляет напрямую (старое поведение).

## Авторизация

- Менеджер входит через `BOT3_ACCESS_CODE` (env переменная на Railway)
- `requireAuth(fn)` — обёртка для всех handler-функций
