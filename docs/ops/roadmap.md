# Roadmap — что в разработке

> Обновлять при завершении или изменении приоритетов. Актуально: июнь 2026.

## В активной разработке

### Полный цикл 15+15 — следующий приоритет (июнь 2026)

Зафиксировано в `docs/core/content-cycle.md`. Реализовать по порядку:

#### 🔲 Шаг 1 — Напоминания Metricool (дни 3, 7, 12)
`checkAnalyticsCycle` уже проверяет подключение. Добавить:
- если `!metricoolConnected && daysSinceWave1 ∈ [3, 7, 12]` → напоминание клиенту в Bot2
- Не более 3 напоминаний

#### 🔲 Шаг 2 — Мини-опрос без Metricool (день 15)
Если Metricool не подключён → Bot2 задаёт 3 вопроса:
1. «Какие посты получили больше реакций — фото, карусели или видео?»
2. «Что вашей аудитории понравилось больше всего?»
3. «Что хотели бы изменить в следующем контенте?»
Ответы → `session.clientFeedback` → используются при генерации Wave2

#### 🔲 Шаг 3 — Claude-анализ как топ-маркетолог (день 15)
Промпт: Claude берёт роль лучшего маркетолога, анализирует:
- Metricool данные (если есть) — охваты, сохранения, реакции по форматам
- Ответы мини-опроса (если нет Metricool)
- Q1-Q12 бизнес-контекст + нишевые знания
Результат → `session.analyticsInsights` → передаётся в block7/block9 как дополнительный контекст

#### 🔲 Шаг 4 — Свежая генерация Wave2
После аналитики запускается:
- block7 (с аналитическими инсайтами в промпте)
- block8 (обложки Wave2)
- block9 (контент-план дни 16–30)
- visual generation Wave2
- Доставка через Bot3 → клиент

#### 🔲 Шаг 5 — Предложение продления (день 30)
Bot2 отправляет клиенту:
- Итог месяца: что было сделано
- Кнопки выбора пакета (тот же / другой тариф)
- Ссылка на оплату Stripe
- Без скидки 20% (только первый месяц)

#### 🔲 Шаг 6 — Месяц 2+: накопленная аналитика
При оплате следующего месяца:
- Анализируется весь доступный период Metricool (не только 15 дней)
- `session.monthNumber++`
- `session.analyticsHistory[]` накапливает все предыдущие анализы
- Claude видит тренды: что улучшалось, что не менялось

**Читать перед реализацией:** `docs/core/content-cycle.md`

### Мультиплатформенные подписи

### Approval flow клиента

- [ ] Клиент видит список 15 постов (предпросмотр)
- [ ] Одобряет каждый
- [ ] Автопубликация через Metricool после одобрения

### Обложки видео

- [ ] Автоматически из промпта первого кадра видео
- [ ] Используется KIE image API (не Veo3)
- [ ] Для Стандарт: 4 обложки, для Профи: 8 обложек

### Библиотека — умное переиспользование контента

**Суть**: когда в библиотеке накопится достаточно материала — переиспользовать вместо генерации новых.

**Что уже есть** (фундамент готов):
- ✅ `video_library/` — хранилище одобренных видео
- ✅ `extractVideoTags()` — Claude извлекает теги из промпта
- ✅ `searchLibrary(tags)` — поиск совпадений по тегам
- ✅ При генерации Bot3 показывает совпадения из библиотеки

**Что добавить позже**:
- [ ] Библиотека изображений (сейчас только видео)
- [ ] Переиспользование видео-фрагментов (~8 сек B-roll) с заменой субтитров — экономия $0.15/фрагмент
- [ ] Порог для автоиспользования: если совпадение > N% тегов → брать из библиотеки без генерации
- [ ] Оценка совместимости ниш (кофейня + пекарня → можно, керамика + бухгалтер → нельзя)

**Приоритет**: после 15+15 цикла и выхода на рынок
