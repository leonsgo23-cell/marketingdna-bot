# Roadmap — что в разработке

> Обновлять при завершении или изменении приоритетов. Актуально: июнь 2026.

## В активной разработке

### 15-дневный цикл (15+15) — РЕАЛИЗОВАН (июнь 2026)

**Суть**: доставка контента в два захода.

**Флоу:**
1. День 1: `deliverVisualPackage` доставляет первую половину (фото/карусели/сторис/видео). Флаги: `wave1DeliveredAt`, `wave2Pending=true`
2. Клиент нажимает "Я опубликовал первый пост" → `postingStartedAt` → запускается аналитический цикл
3. День 15: `checkAnalyticsCycle` (раз в час) → Metricool analytics → Claude анализирует → генерирует коррекции → менеджер получает в Bot3
4. Менеджер нажимает "Отправить клиенту" → текстовые коррекции идут клиенту + создаётся `.wave2.trigger`
5. `checkTriggers` подхватывает `.wave2.trigger` → `deliverVisualPackage` (доставляет вторую половину)

**Триггеры:** `.wave2.trigger` — создаётся bot3.js при одобрении коррекций

**Сессия клиента:** `wave1DeliveredAt`, `wave2Pending`, `wave2DeliveredAt`, `postingStartedAt`, `analyticsCycles`

**Если Metricool не подключён:** запрашиваем скриншоты у клиента на день 14/15

### Мультиплатформенные подписи

- [ ] Q5 в Bot2: кнопки выбора платформ (Instagram / TikTok / Facebook / LinkedIn)
- [ ] Генерация разных вариантов подписей под каждую платформу
- [ ] `session.platforms` — массив выбранных платформ

### Approval flow клиента

- [ ] Клиент видит список 15 постов (предпросмотр)
- [ ] Одобряет каждый
- [ ] Автопубликация через Metricool после одобрения

### Обложки видео

- [ ] Автоматически из промпта первого кадра видео
- [ ] Используется KIE image API (не Veo3)
- [ ] Для Стандарт: 4 обложки, для Профи: 8 обложек

### Библиотека — умное переиспользование контента

**Суть**: когда в библиотеке накопится достаточно материала — переиспользовать вместо генерации новых.

**Что уже есть** (фундамент готов):
- ✅ `video_library/` — хранилище одобренных видео
- ✅ `extractVideoTags()` — Claude извлекает теги из промпта
- ✅ `searchLibrary(tags)` — поиск совпадений по тегам
- ✅ При генерации Bot3 показывает совпадения из библиотеки

**Что добавить позже**:
- [ ] Библиотека изображений (сейчас только видео)
- [ ] Переиспользование видео-фрагментов (~8 сек B-roll) с заменой субтитров — экономия $0.15/фрагмент
- [ ] Порог для автоиспользования: если совпадение > N% тегов → брать из библиотеки без генерации
- [ ] Оценка совместимости ниш (кофейня + пекарня → можно, керамика + бухгалтер → нельзя)

**Приоритет**: после 15+15 цикла и выхода на рынок
