# Путь клиента — Marketing DNA

## Бесплатный пакет

```
1. Клиент находит Marketing DNA (сайт / реклама)
2. Вводит /start в Bot2
3. Заполняет анкету (5-7 минут) — handleStart() + handleMessage() в bot2.js
4. Bot2 пишет {id}.trigger → checkTriggers() в index.js подхватывает
5. Bot1 генерирует бесплатный пакет (block_free_package.js):
   - Контент-план 7 дней
   - SEO-статья (образец)
   - Сценарий ролика
   - Сценарий карусели
   - Пример обложки
   - 1 реальное AI-фото
6. Bot3: менеджер видит всё → нажимает "Одобрить" → {id}.free_approved.trigger
7. Клиент получает в Bot2:
   - Текстовый пакет
   - 1 AI-фото
8. Сразу предлагается скидка 20% (48 часов) на платный пакет
```

## Платный пакет

```
1. Клиент нажимает Stripe-ссылку из Bot2 (после бесплатного или напрямую)
2. Stripe оплата → webhook → {id}.paid_init.trigger → startPaidOnboarding()
3. ПРОВЕРКА: проходил ли клиент бесплатный флоу?
   ├── ДА (есть freeQ2 + freeQ1 + contentLanguage) → сразу к уточняющим вопросам
   └── НЕТ (прямая покупка) → сначала 3 вводных вопроса:
       а) Город/регион
       б) Сайт (URL → Jina AI читает автоматически) или текстовое описание
          ⚠️ Instagram не запрашивается — нельзя прочитать автоматически
       в) Язык контента (кнопки 🇷🇺/🇱🇻/🇬🇧)
4. Bot2 задаёт уточняющие вопросы Q1-Q12 (startPaidOnboarding в index.js):
   - Формат видео (Тип А "человек в кадре" / Тип Б "AI B-roll")
   - CTA (директ с магнитом / только директ / без директа)
   - Лид-магнит (если direct_magnet)
   - Цель контента (новые клиенты / существующая аудитория)  ← ОБЯЗАТЕЛЬНЫЙ вопрос
   - Язык аналитики
   - Язык контента
   - Ссылки на соцсети
4. Bot1 запускает генерацию (блоки 1-9), ~15-20 минут
5. visual.js генерирует изображения и видео параллельно
6. Bot3: менеджер проверяет ВСЁ → одобряет → {id}.approved.trigger
7. Клиент получает в Bot2 полный пакет через deliverVisualPackage()
```

## Доп. язык (обновлено 20.06.2026)

```
1. Bot2 предлагает доп. язык после получения основного пакета
2. Клиент оплачивает → Stripe → .addlang_{lang}.trigger
   (тест без Stripe: /test_addlang {chatId} {lang} в Bot3)
3. Запускается runTranslationJob() — 3 шага:

   ШАГ 1 — Локальное исследование (searchLocalMarketPhrases):
   - Tavily ищет в целевом языке: реальные отзывы, форумы, поисковые запросы
   - Sonnet строит "локальное семантическое ядро": ключевые слова, голос покупателя,
     местные боли, темы для статей — всё на целевом языке
   - Результат: localSemanticData — основа для всего дальнейшего

   ШАГ 2 — SEO-статьи (generateFreshSeoArticles):
   - Строятся на localSemanticData (реальные ключевые слова целевого рынка)
   - НЕ перевод основного семантического ядра — оригинальный контент для местного SEO
   - Sonnet пишет как носитель языка для местной аудитории

   ШАГ 3 — Адаптация маркетингового контента (Sonnet, не Haiku):
   - Карусели, фото, сторис, видео, обложки, контент-план
   - Контекст: бизнес-профиль + аудитория + кастдев-фразы + localSemanticData
   - НЕ "переведи" — "адаптируй как местный маркетолог-носитель языка"
   - Технические маркеры (ВИДЕО, СЦЕНА, КАРУСЕЛЬ и т.д.) всегда на русском
   - Видео субтитры: ffmpeg пересобирает с новым текстом

4. Менеджер проверяет адаптированный контент в Bot3 → одобряет → клиент получает
```

**Работает для любого языка**: ru → lv, ru → en, lv → en и т.д.
**При третьем языке**: тот же 3-шаговый пайплайн применяется повторно.

## Важные детали UX

- Клиент ждёт "15-20 минут" — НЕ менять, создаёт ощущение персональной работы
- Менеджер проверяет ВСЁ перед отправкой (до 20-30 стабильных клиентов)
- Контент-план — всегда ДВЕ волны по 15 дней: Волна 1 (привлечение) + Волна 2 (активация). Q9 влияет на стратегию обоих.
- На 15-й день — аналитика Metricool, затем доставка Волны 2 клиенту
- Сезонность учитывается автоматически в блоке 9
