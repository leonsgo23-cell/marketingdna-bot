## Карусели

- Формат скриптов: `КАДР N: / Текст поверх фото: / Подпись к посту:` (каждый слайд = мини фото-пост)
- Хук на изображении: max 6 слов (через `wordSlice()`)
- Подпись: Telegram caption под слайдом, отдельно от изображения
- Количество слайдов: 7
- Полоса: `rgba(0,0,0,0.45)` — полупрозрачная (НЕ 0.72 — слишком темно)

## Видео

- `maxVideos=1` в тестах (всегда, пока нет явного другого разрешения)
- Длина: 30 сек (обрезка через `ffmpeg -t 30`) — БЕЗ ffprobe (не работает в Railway)
- Формат: ТОЛЬКО 9:16 вертикаль (Reels/Shorts/TikTok)
- YouTube = только Shorts (горизонтальные видео вне сервиса)
- **Фрагменты: ровно 4** (4 × 8 сек = 32 сек сырого материала → обрезается до 30 сек). Меньше 4 = итоговое видео короче 25 сек — нарушение.

## Визуал — правила для всех пакетов (free + paid + visual_sample)

- Все изображения генерируются через `startImage` — функция автоматически: 1) очищает промпт от текстовых инструкций (`stripTextFromPrompt`), 2) добавляет реалистичный суффикс. Текст на изображении — ТОЛЬКО через `overlayTextOnImage`, не через Kie.ai.
- Карусель отправляется менеджеру как альбом (sendMediaGroup), после — отдельное сообщение с кнопками по каждому слайду.
- Под каждым изображением три кнопки: 🔄 Переделать / ✏️ Изм. текст / 🚫 Без текста
- После карусели — отдельное сообщение с подписями к постам (Подпись к посту из скрипта)
- Сторис: текст всегда внизу (position = 'bottom')

## visual_sample — порядок доставки

- **Карусель**: все слайды генерируются, потом приходят одним альбомом (медиа-группа). После альбома — отдельное сообщение с кнопками 🔄/✏️ для каждого слайда.
- **Фото, обложка, сторис**: каждый приходит сразу как готов.
- **Видео**: сначала приходит полное 30-секундное видео с кнопками ✏️ Хук / ✏️ Тема / ✏️ CTA / 🔄 Переделать всё. Потом — 4 фрагмента с кнопками только ✅ Ок / 🔄 Переделать (без текстовых кнопок на фрагментах).

## Стиль изображений

- **Всегда реальная фотография** — независимо от ниши клиента
- Арт-студия, галерея, дизайн-бюро — это контекст **что снимать**, не **как выглядит изображение**
- Никакого живописного, рисованного, иллюстративного стиля — даже если бизнес связан с искусством
- Суффикс в `startImage` запрещает artistic style явно + объясняет что бизнес-контекст не влияет на стиль съёмки

## Контент

- Цель контента (новые клиенты / существующая аудитория) — ОБЯЗАТЕЛЬНЫЙ вопрос в анкете
- Всегда ОДИН контент-план (не два)
- Никогда не писать "AI-сервис" — только конкретное название бизнеса клиента
- Сезонность учитывается автоматически

## Доставка

- Клиент НИКОГДА не получает контент без одобрения менеджера в Bot3
- Клиент ждёт "15-20 минут" — НЕ менять формулировку

## Пакеты

- Старт Плюс / Пакет Б — НЕ СУЩЕСТВУЕТ, упразднён
- Все суммы только в EUR/USD, рубли никогда

## Технические запреты (проверено — не работает в Railway)

- НЕ использовать `@napi-rs/canvas fillText` — fontconfig не работает в Railway nixpacks
- НЕ использовать Sharp + SVG `<text>` — librsvg не рендерит TTF шрифты
- НЕ загружать Inter-Bold.ttf из интернета — однажды вернуло HTML 404
- НЕ использовать `subtitles=` фильтр с SRT — требует fontconfig
- НЕ использовать `slice(0, N)` для обрезки текста — режет посередине слова
- НЕ использовать ffprobe — не работает в Railway

## Изображения в delivery — локальные файлы приоритетнее URL (июнь 2026)

И `deliverFreePackage`, и `deliverVisualPackage` используют локальные файлы как основной источник. URL Kie.ai — только запасной вариант (истекают через 24-72ч).

- Free: `bestMedia(localPath, urlFallback)` — проверяет `_ov.jpg` → raw → URL
- Paid: `buildMediaArray(urls, localPaths)` — строит массив из localPaths с fallback на URL

**НЕ возвращаться к прямым URL** — это намеренное решение для надёжности.

## Metricool — анонимная ссылка (июнь 2026)

Клиент подключает Instagram **без регистрации** в Metricool через одноразовую ссылку.

**API эндпоинт:**
```
POST https://app.metricool.com/api/v2/settings/brands/connections/anonymous-link
     ?userId={METRICOOL_USER_ID}&blogId={clientBlogId}
```
**Ответ:** `{ data: { link: "https://f.mtr.cool/XXXXX", expiresAt: { dateTime, timezone } } }`  
**Срок действия:** 71 час.

Вызывается в `analytics_yes` обработчике (bot2.js) — только когда клиент сам согласился на аналитику.  
**НЕ вызывать при оплате** — бренд создаётся только по желанию клиента.

## VISUAL_BASE_URL — всегда с https:// (июнь 2026)

В `site_builder.js` при построении URL всегда добавляется протокол:

## Бесплатный флоу — без анализа конкурентов (11.06.2026)

`runBlock3` (Tavily, анализ конкурентов) НЕ вызывается в бесплатном флоу.
Бесплатный пакет генерируется только на основе `buildReturningProfiles` (бизнес-профиль + аудитория из 4 вопросов клиента).
Конкуренты — только в платном флоу.
В DEMO-триггере (`*.demo.trigger`) block3 остаётся — это внутренний демо-пакет для продаж.

## Платный флоу — конкуренты из Q3 автоматически (12.06.2026)

`handleReturningChoice` в `block0_returning.js`:
- Если Q3 (`paidAnswers.competitors`) заполнен → `extractCompetitorsFromQ3()` → `session.competitorNames` → Block3 скрейпит реальные сайты через Tavily
- Если Q3 пустой/«не знаю» → `autoSearchCompetitors = true` → Block3 через Claude описывает типичных конкурентов ниши (без Tavily)
- Ручной ввод конкурентов от администратора УБРАН — данные берутся из анкеты клиента

`index.js` RETURNING_CHOICE case: при `choice === true && session.step === DONE` запускает полную цепочку блоков 4→5→3→6→7→8→9.

## Block9 — всегда две волны по 15 дней (12.06.2026)

Block9 всегда генерирует два плана:
- Волна 1 (дни 1–15): привлечение и доверие
- Волна 2 (дни 16–30): активация и продажи

Q9 цель клиента (`session.monthlyGoal`) влияет на стратегию ОБОИХ планов — больше не разветвляет на "один план vs два".
Финальное сообщение напоминает: на 15-й день — аналитика Metricool, потом доставка Wave2.

## Бесплатный пакет — Claude-анализ конкурентов ниши (12.06.2026)

В index.js после `buildReturningProfiles` для бесплатного триггера добавлен лёгкий вызов Claude (без Tavily):
- Анализирует типичных конкурентов ниши из знаний модели
- Результат → `enrichedData.competitorBrief` → `block_free_package.js`
- `block_free_package.js` использует `compSection` в промптах контент-плана, статьи и карусели
- Объём: 4–5 предложений, ~600 токенов, ~30 сек

## Block9 — состав каждой волны (12.06.2026)

Весь пакет делится пополам между Волной 1 и Волной 2. Каждая волна получает ровно половину:

| Тип контента | Старт | Стандарт | Профи |
|---|---|---|---|
| Карусели | 4+4 | 4+4 | 4+4 |
| Фото-посты | 4+4 | 4+4 | 4+4 |
| Stories | 7-8+7-8 | 7-8+7-8 | 7-8+7-8 |
| Видео B-roll | нет | 2+2 | 4+4 |
| Обложки | нет | 2+2 | 4+4 |

Block9 передаёт `waveContentInstruction` в оба промпта — Claude строит расписание с учётом всех форматов волны.

## Библиотека контента — архитектура и решения (12.06.2026)

**Где хранится:** Railway volume `/root/.marketingdna-client-sessions/video_library/` и `photo_library/`

**Три цели библиотеки:**
1. Повторное использование фото/видео для других клиентов схожих ниш
2. Отслеживание что уже отправлялось конкретному клиенту (не повторять)
3. Самообучение через фидбек менеджера (`prompt_learning.js`)

**Что работает:**
- Сохранение в библиотеку при одобрении менеджером (`/save_approved_content`) — для free и paid ✅
- Повторное использование фото в **бесплатном** пакете (`tryPhotoLibrary` перед Kie.ai) ✅
- Исключение уже отправленного контента при поиске — `usedPhotoIds/usedVideoIds` в `content_history/{chatId}.json` ✅
- Самообучение промптов — `prompt_learning.js`, уроки добавляются в каждый промпт ✅

**Что НЕ включено (сознательно):**
- Повторное использование фото в **платном** пакете — ждём критической массы (100+ фото, 20+ видео)
- Команда `/history {chatId}` в Bot3 — история клиента смотрится через Google Sheets
- Автоматическое повторное использование видео — только информационное уведомление менеджеру

**Исправленный баг (12.06.2026):** при сохранении платных фото в библиотеку передавался пустой промпт → теги были бесполезны → поиск не работал. Исправлено: `save_approved_content` читает `visual.json` и передаёт реальные промпты через `extractByPrefix`.

**Критическая масса:** смотреть `/library` в Bot3. При 100+ фото и 20+ видео — включать повторное использование для платного пакета.
