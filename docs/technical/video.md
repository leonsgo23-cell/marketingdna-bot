# Видео — техническая реализация

> Финальная реализация: 02.06.2026. Обновлено: 24.06.2026. Не менять без явного подтверждения.

## Кто получает видео

- **Старт** — AI-видео НЕ генерируется. Только 8 сценариев "человек в кадре" как подарок.
- **Стандарт** — 4 B-roll видео (Kie.ai Veo3)
- **Профи** — 8 B-roll видео + 8 обложек Highlights

## Формат ТЗ для видео (block7_scripts.js, обновлено 15.06.2026)

Block7 пишет для каждого видео 4 сцены в двух языках:

```
ВИДЕО N: [тема — макс 35 символов]
Настроение: [тип]
Эмоция зрителя: [хук — макс 35 символов]

СЦЕНЫ ДЛЯ ГЕНЕРАЦИИ:
СЦЕНА 1 (0-8 сек):
  EN: [конкретный EN prompt для Veo3 — детали этого бизнеса, vertical 9:16, no text]
  RU: [что видит зритель — для проверки менеджером]
СЦЕНА 2 (8-16 сек):
  EN: [...]
  RU: [...]
СЦЕНА 3 (16-24 сек):
  EN: [...]
  RU: [...]
СЦЕНА 4 (24-30 сек):
  EN: [...]
  RU: [...]
```

EN-строки → Veo3. RU-строки → менеджер видит в Bot3 до генерации.

## Пайплайн генерации (generateOneVideo в visual.js)

```
1. splitScriptToScenes(videoScript):
   — Primary: извлекает 4 "СЦЕНА N: EN: ..." строки напрямую (без Haiku)
   — Fallback: Claude Haiku генерирует сцены (для старых скриптов без СЦЕНА-блоков)
2. startVideo(prompt) → Kie.ai Veo3 API
3. pollTask() — ждём до 7 минут каждый фрагмент
4. Скачиваем .mp4 фрагменты → ffmpeg concat → merged.mp4 (rawPath — БЕЗ текста)
5. buildTimedSrt + addTimedSubtitles → final.mp4 (localPath — С текстом)
6. Сохраняем в results.json: rawPath, localPath, subtitleText, hookText, themeText, ctaText
```

**Важно**: `rawPath` всегда БЕЗ субтитров — база для редактирования текста. `localPath` — финальный файл С субтитрами.

## Превью сценариев для менеджера (15.06.2026)

Перед началом видео-генерации `notifyBot3VideoScriptsPreview` отправляет менеджеру в Bot3 карточку:
- Название каждого видео
- Хук (Эмоция зрителя)
- RU-описание каждой из 4 сцен

Менеджер видит ЧТО будет в кадре до того как $1.20 потрачено.

## Три блока текста поверх видео

| Блок | Тайминг | Лимит | Источник текста |
|------|---------|-------|----------------|
| **Хук** | 0–4 сек | до 35 символов (1 строка) | "Эмоция зрителя:" из скрипта |
| **Тема** | 35%–65% видео | до 35 символов (1 строка) | Заголовок сценария, max 5 слов |
| **CTA** | Последние 8 сек → до конца | до 66 символов (3 строки × 22) | `ctaPreference` клиента |

**CTA идёт до реального конца видео** (`ctaEnd = 9999` в SRT), независимо от измеренной длины.

## Техническая реализация текста

- Одна тёмная полоса снизу: `drawbox=...color=black@0.72:t=fill` + текст
- Шрифт: `Inter-Bold.ttf`, размер 52px, 1 строка = ~22 символа при 1080px ширине
- `_splitLines(text, 22)` — автоперенос по словам, максимум 3 строки
- `_buildDrawtextBlock(text, start, end, baseTmpPath)` — строит ffmpeg drawtext фильтры с тайммингом `enable='between(t, start, end)'`
- `buildTimedSrt(hook, cta, duration, theme)` → SRT-контент для `addTimedSubtitles`

## Что НЕЛЬЗЯ делать (проверено, не работает)

- `subtitles=` фильтр с SRT — требует fontconfig (нет в Railway nixpacks)
- `slice(0, N)` для обрезки текста — режет посередине слова
- ffprobe для получения длины видео — не работает в Railway

## Регенерация видео

- `regenVideoFromScript(clientChatId, videoScripts, feedback)` — полная перегенерация
- `regenVideo(clientChatId, videoIndex, feedback)` — перегенерация одного видео (берёт `fragPaths` из results.json, не CDN-ссылки)
- `regenSubtitle(clientChatId, videoIndex, newSubtitleText)` — только субтитры

### regenSubtitle — логика (24.06.2026)

**Вызывается**: из `/regen_video` endpoint когда есть `subtitleOverride` в теле запроса.

**Входные данные**: `newSubtitleText` = строка с одним или несколькими полями:
- `"Хук: текст"` — только хук (остальные берутся из сохранённых)
- `"Тема: текст"` — только тема
- `"СТА: текст"` — только CTA
- Все три строки сразу

**Парсинг + fallback**: для каждого поля:
- Если поле найдено в тексте → берётся новое значение
- Если нет → берётся из `videoData.hookText / themeText / ctaText` (сохранены в results.json)

**Лимиты**: hookText ≤ 35, themeText ≤ 35, ctaText ≤ 66 символов (обрезаются автоматически)

**Наложение**: `buildTimedSrt` + `addTimedSubtitles(rawPath, ...)` — тайминговый (не статичный).

**Сохранение**: обновляет `results.json` с новыми `hookText`, `themeText`, `ctaText`, `localPath`.

**После**: вызывает `notifyBot3SingleVideo` (кнопки по полям + переснять + без текста).

### Флоу редактирования текста видео в Bot3 (24.06.2026)

После получения видео менеджер видит сообщение с кнопками:
```
[✏️ Изменить хук]  [✏️ Изменить тему]  [✏️ Изменить CTA]
[✏️ Изменить всё]  [🔄 Переснять сцену]
[🎬 Версия без текста]
```

- **Изменить хук** — спрашивает только хук, тема и CTA сохраняются
- **Изменить тему** — спрашивает только тему, хук и CTA сохраняются
- **Изменить CTA** — спрашивает только CTA, хук и тема сохраняются
- **Изменить всё** — просит три строки сразу (`Хук: ... \n Тема: ... \n CTA: ...`)
- **Версия без текста** — присылает `rawPath` (видео без субтитров) для самостоятельного наложения
- **Переснять сцену** — запускает Veo3 регенерацию одной или нескольких сцен
